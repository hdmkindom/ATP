### 对这次 `MulChar.sum_eq_zero_of_ne_one` 而言：不是主因

原因很直接：

- 我已经在本地 `mathlib` 源码里找到了该 theorem：

所以：

- “本地版本太老，导致这个 theorem 根本不存在”这一解释，不适用于本轮的这个具体错误。

### 但 LeanSearch 的结果确实不是按你的本地环境版本精确锁定的

ATP 当前用的 ax-prover `search_lean` 实现，默认直接请求：

- `https://leansearch.net/search`

本地实现见：

- `ax_prover/tools/lean_search.py`

其中默认 URL 是 `https://leansearch.net`。

此外，LeanSearch 公开仓库里可以看到：

- `reaslab/LeanSearch` README 提到公开数据集是 `mathlib_informal_v4.16.0`
- `frenzymath/LeanSearch` README 明确提醒：若索引时 Lean/toolchain 不匹配，会出现版本相关问题

因此，更准确的说法是：

- LeanSearch 返回的是**远端索引中的候选结果**；
- 它不是你本地 `.lake/packages/mathlib` 的“严格同版本、严格同导入、严格可直接调用”的证明助手；
- 搜到以后仍然需要做本地 import 和类型校验。

这里需要强调：  
对本轮 `MulChar.sum_eq_zero_of_ne_one` 来说，我更倾向于判定“**适用性误判**”而不是“**版本过旧**”。
